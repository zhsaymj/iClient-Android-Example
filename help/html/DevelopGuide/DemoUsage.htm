Demo简介

发送反馈

SuperMap iClient for Android提供的Demo示范代码位于【SuperMap iClient for Android 产品目录】\samples\iclient-android-demo，同级目录的iclient_android_demo_6.1.4_*.apk是相应的可直接运行于Android设备的应用程序。

关于Demo示范代码的运行和使用方法，请参考第五步 编译并运行。

备注：导入iclient-android-demo工程后，Eclipse环境还需进行如下设置，否则编译失败：选择菜单“Window->Preferences”的“General->Workspace”的“Text file encoding”设置为“UTF-8”，“New text file line delimiter”设置为“Unix”。

运行示范代码后可以看到Demo提供的基本功能，与在Android设备上运行iclient_android_demo_6.1.4_*.apk看到的效果是一样的。启动iClientAndroidDemo后，在弹出的对话框输入可访问的地图服务地址，会弹出Demo菜单，可以看到包含以下功能：

  • 地图浏览

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

将按照程序启动时指定的地图服务进行浏览，支持缩放、平移等操作。

  • 地图切换

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，依次在管理菜单中点击地图切换 > 添加新服务，在弹出的对话框中输入服务地址（http://serverIP:8090/iserver/services）并确定，会弹出服务器上的所有地图服务列表，选择并长按要切换的地图，在弹出的对话框中可以进行三种操作：

1. 切换当前地图服务——即显示选择的地图服务

2. 编辑当前地图服务——即编辑选择的地图服务的地址

3. 删除当前地图服务——即在当前显示的地图服务列表中删除选择的地图服务，不会影响服务器上的服务列表

  • 注记浏览

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

浏览Demo示范代码里写出的注记，本例中包括一个点标识和文本注记，显示为“北京超图”。

  • 注记添加

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，在地图上的任意位置长按，会弹出当前点的坐标和四种颜色的注记，选择注记后就在地图上添加了相应的注记。

  • 距离查询

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，在地图上的任意位置长按，弹出的对话框会显示当前位置，可输入查询的距离范围，可点击下拉框选择查询图层的名称。

  • 地图更新

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，程序会将添加到demo上的图层缓存删除，不会删除底图。

  • 地图叠加

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，依次在管理菜单中点击地图叠加 > 添加地图服务，在弹出的对话框中输入服务地址（http://serverIP:8090/iserver/services）并确定，会弹出服务器上的所有地图服务列表，选择并长按需要叠加的地图，弹出添加当前地图服务的按钮，点击即可将选择的地图添加到地图视窗中。地图叠加功能支持动态投影，会将待添加的地图动态投影到原有地图相同的坐标系下显示。

  • 绘点线面

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

点击进入后，点击菜单进行相关操作，如绘点、绘线、绘面、清除、浏览、结束。

  • 可视浏览

Demo使用的地图实例地址为 /map-china400/rest/maps/China。

在设置了固定比例尺的情况下进行进图浏览。

  • 交通换乘

Demo使用的地图实例地址为 /map-changchun/rest/maps/长春市区图。

点击进入后，在弹出的对话框输入可访问的服务地址和交通换乘实例地址（交通换乘实例和地图服务的数据必须保持一致，如：/traffictransferanalyst-sample/restjsr/traffictransferanalyst/Traffic-Changchun）。点击移动设备的菜单按钮，点击“路线搜索”菜单，在弹出的对话框中输入要查询的起点和终点，用户依据个人喜好，选择换乘方式“最少时间”、“最短距离”、“少步行”、“少换乘”等，如下图：

上图输入的站点依据“最少时间”分析结果如下图，得到所有的路线查询结果，展开各路线结果，详细列出路线信息，也可点击“查看地图”按钮，返回地图视窗中，查看具体的查询路线。

  • 网络分析

Demo使用的地图实例地址为 /map-changchun/rest/maps/长春市区图。

点击进入后，在弹出的对话框输入可访问的服务地址和网络分析实例地址（网络分析实例和地图服务的数据必须保持一致，如：/transportationanalyst-sample/rest/networkanalyst/RoadNet@Changchun）。点击移动设备的菜单按钮，可选择“最佳路径分析”、“旅行商分析”、“服务区分析”、“最近设施查找”、“多旅行商分析”、“选址分区”等菜单，弹出相应分析对话框，选取分析站点进行分析，分析结果在地图中高亮显示。

最佳路径分析结果如下图，浅蓝色圆点表示站点，最佳路径分析的结果路线以蓝色高亮表示。

最近设施分析结果如下图，橙色气泡表示设施点，浅蓝色圆点表示事件点，事件点到最近设施的分析结果路线以蓝色高亮表示。

多旅行商分析结果如下图，橙色气泡表示配送中心点，浅蓝色圆点表示配送目的地，各个中心点配送的分析结果路线以蓝色高亮表示。

  • 地物编辑

Demo使用的地图实例地址为 /map-jingjin/rest/maps/京津地区土地利用现状图。

点击进入后，在弹出的对话框输入可访问的服务地址（和Demo启动后第一次输入的服务地址必须保持一致）和数据分析实例地址（数据分析实例和地图服务的数据必须保持一致，如：/data-jingjin/rest/data）。点击移动设备的菜单按钮，点击“地物编辑”，弹出“地物编辑”对话框。数据编辑的图层为“Landuse_R@Jingjin”。

  • 选择地物后方可进行编辑地物、编辑属性和删除地物操作。
  • 添加和编辑地物操作后还需要点击“提交”按钮完成当前编辑操作。
  • 编辑地物支持平移地物、增加节点（在线上点击则增加了节点）、拖拽节点、删除节点（双击待删除的节点则删除了此节点）。

  • 选中某对象后，弹出选中对象属性列表，长按待编辑的字段，选择修改当前属性数据，修改属性后，点击“提交更新”则完成属性编辑操作。

    • 数据查询

    Demo使用的地图实例地址为 /map-world/rest/maps/World。

    点击进入后，在弹出的对话框输入可访问的服务地址（和Demo启动后第一次输入的服务地址必须保持一致）和数据分析实例地址（数据分析实例和地图服务的数据必须保持一致，如：/data-world/rest/data）。点击移动设备的菜单按钮,可选择“ID查询”、“几何查询”、“缓冲区查询”、“SQL查询”等菜单。

  • ID查询的查询图层为“Countries@World”，查询IDs为[110, 239]，查询结果在地图中高亮显示。”
  • SQL查询的查询图层为“Countries@World”，查询条件为"SMID = 240"，查询结果在地图中高亮显示。
  • 几何查询的“Countries@World”,可绘制点、线、面分别进行查询，查询结果的所有对象也都在地图中高亮显示。

  • 缓冲区查询的查询图层为“Capitals@World”，缓存半径为30度，可绘制点、线、面分别进行查询，查询结果的所有对象也都在地图中高亮显示。
