使用MBTiles离线数据包说明

发送反馈

地图浏览时支持使用遵循 MBTiles 地图瓦片存储规则的 MBTiles 离线包。

1.使用MBTiles离线包作为动态图层的缓存

使用MBTiles离线包进行地图浏览时，需首先使用SuperMap iServer Java 的分布式切图功能生成，具体实现方法可参照iServer帮助文档 > 配置管理 > 二维地图缓存的配置 相关内容；其次数据包的命名需符合“地图名.mbtiles”规则，如China.mbtiles、长春市区图.mbtiles；之后需要将生成的 MBTiles 数据包拷贝到移动端“\mnt\sdcard\supermap\sqlite”目录下，若没有“supermap\sqlite”文件夹，请自行创建或启动apk浏览一次后自动创建。

用户按照数据包中包含的切图比例尺进行地图浏览时系统会自动优先读取MBTiles离线包文件缓存，可完全不依赖于任何网络通讯进行快速浏览地图。

2.MBTiles离线数据图层

MBTiles离线数据图层，在移动端可以直接读取MBTiles离线数据，从而在没有网络离线状态下浏览和操作图层。 构建MBTiles离线数据图层时需要MBTiles离线数据包所在移动端的路径，该路径可以是绝对路径也可以是相对于内置sdcard的一个子目录（如数据在移动端的"mnt/sdcard/supermap/China.mbtiles"中，那么创建图层需要的参数即为"supermap/China.mbtiles"），详见地图图层的介绍。

支持符合含MBTiles规范的命名MBTiles后缀格式的数据，也可以支持对MBTiles格式扩展的格式SMTiles的数据。
